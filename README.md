## 🚀 The Mission: Why MQLC?

Traditional educational institutions often struggle with **Paper Friction**—manual registers, lost enrollment forms, and zero analytical insight into student demographics. 

**MQLC** was born to solve this. It transitions a Madrasa from offline chaos to a professional **Cloud-Native Platform**. It isn't just a website; it’s a lifecycle management tool that bridges the gap between traditional values and modern administrative efficiency.

---

## 💡 The Journey: From Idea to Reality

### 1. Ideation Phase
The goal was simple: **Professionalism at first glance.** We wanted a platform that felt as premium as a high-end SaaS but remained accessible to a diverse, multi-lingual community.

### 2. Development Phases (The Agile Sprint)
The project was brought from a blank canvas to a fully operational administrative suite in approximately **30 hours** of focused engineering.

*   **Phase I: Resonance & Reach** – Building the multi-lingual landing page with RTL (Urdu) support.
*   **Phase II: The Gateway** – Developing the 3-step registration portal with Aadhar validation and secure PIN generation.
*   **Phase III: The Command Center** – Creating the Admin Dashboard with real-time student matrix and lifecycle tracking (Edit/Delete).
*   **Phase IV: Intelligence** – Integrating Chart.js for data-driven insights and professional export tools.

## 🛠️ Feature Showcase

### 🌐 Public-Facing Website

- [x] **Smart Multi-Lingual Engine:** Zero-dependency language switcher supporting English, Hindi, Marathi, and Urdu with automatic RTL (Right-to-Left) layout adaptation for Urdu content.

- [x] **Localized Numeral Rendering:** All numerical data — stat counters, prayer times, quiz scores — dynamically renders in the user's chosen script (Devanagari, Urdu numerals) for an authentic native experience.

- [x] **Dynamic Bulletin Board:** A horizontally-scrollable carousel that auto-populates with posters, notices (PDF), and quiz cards directly from Cloudinary — no code changes needed to publish new content.

- [x] **Lightbox Media Viewer:** Clicking any poster opens a full-resolution modal with a secure blob-based download button, avoiding raw Cloudinary URL exposure to end users.

- [x] **Live Jamat (Prayer) Timings:** Prayer schedule pulled from Supabase with an astronomical override for Maghrib using the Aladhan API, ensuring sunset-accurate timings for the local city.

- [x] **Quranic Audio Recitation:** A floating audio player with lazy-loaded Surah Al-Alaq recitation (Shaikh Maher Al Muaiqly), supporting play/pause toggle with animated audio bars.

- [x] **Scroll-Driven Animations:** IntersectionObserver-powered reveal effects (fade, slide-left, slide-right) and parallax layers creating a dynamic, immersive browsing experience.

- [x] **Animated Stat Counters:** KPI counters (Students, Programs, Fee, Duration) animate with eased cubic interpolation on scroll — pulling live values from the database rather than hardcoded numbers.

- [x] **Stealth Admin Access:** Triple-clicking the navbar logo redirects to the admin panel — no visible admin link exposed to the public.

### 📝 Digital Registration Portal

- [x] **OTP-Gated Secure Registration:** Parents must enter a 6-digit PIN (generated by admin) before accessing the registration form, preventing spam submissions and unauthorized access.

- [x] **3-Step Wizard Registration:** A progressive multi-step form (Course → Identity → Education) with a visual progress bar, field validation, and conditional logic (e.g., school fields hide for pre-school children).

- [x] **Real-Time Aadhar Validation:** Auto-formatting Aadhar input as `XXXX-XXXX-XXXX` with digit-only enforcement, preventing data-entry errors at the source.

- [x] **Auto-Generated Form Numbers:** Each registration is assigned a unique form number (e.g., `MQLC-26-A1B2`) generated from the current year and a random alphanumeric sequence — no manual numbering.

### 🧠 Interactive Quiz System

- [x] **JSON-Driven Quiz Engine:** Quizzes are defined as simple JSON payloads (topic, questions, options, answers) uploaded to Cloudinary — the quiz UI renders entirely from the JSON, requiring zero code changes.

- [x] **Shareable Quiz URLs:** Clean `?q=filename` parameter approach enables sharing quizzes via WhatsApp or social media without exposing long Cloudinary paths or sensitive session data.

- [x] **Multi-Lingual Quiz Support:** Quiz topics, questions, and options support per-language translations via object-based JSON fields, with automatic script-appropriate numeral rendering.

- [x] **Auto-Expiring Quizzes:** Quizzes automatically hide from the bulletin board after their submission deadline passes, keeping the UI clean without manual intervention.

- [x] **Persistent Leaderboard:** Quiz scores are stored in Supabase with duplicate-submission prevention per phone number, powering a ranked leaderboard visible in the admin panel.

### 🏛️ Admin Command Center

- [x] **Supabase-Authenticated Dashboard:** Email/password authentication with session persistence, automatic token refresh, and clean login/logout state transitions.

- [x] **Pill-Based Sub-Navigation:** A horizontal floating pill-nav system within the Student Management tab (Registration → Manual Entry → Dashboard → Fee Tracker → Analytics) enabling rapid context switching without page reloads.

- [x] **Parent Entry Gateway:** Admins generate unique 6-digit PINs that parents use to access the registration form — creating a controlled enrollment pipeline with full audit trail.

- [x] **Pending Approval Queue:** New registrations land in a review queue where admins can inspect full application details, assign a monthly fee, and approve or reject with a single action.

- [x] **Manual Student Entry:** A legacy override form for backdated entries or edge cases where the OTP registration flow isn't feasible, with all fields mirroring the parent registration form.

- [x] **Dynamic Student Matrix:** A searchable, filterable student list grouped by batch (Zuhr, Asr, Maghrib) with real-time text search, status/batch/course dropdown filters, and an automated "Approved" default view that hides rejected entries unless explicitly requested.

- [x] **Inline Student Editor:** Each student row has an edit button that opens a modal dialog for modifying name, batch, course, and lifecycle status — changes reflect immediately in the matrix and KPI cards.

- [x] **Student Lifecycle Management:** Four-state tracking (Approved → Left, Pending → Rejected) with visual badge indicators and filter-aware rendering across the entire dashboard.

### 💰 Fee Tracker

- [x] **Per-Student Fee Configuration:** Each student's monthly fee is individually set at approval time, supporting variable pricing for scholarships, different courses, or family-based concessions.

- [x] **Month-by-Month Navigation:** Previous/Next month buttons with a live label showing the selected month, enabling admins to record or review payments for any month — past or future.

- [x] **Partial Payment Support:** The system supports multiple partial payments per student per month (e.g., ₹300 now, ₹200 later), aggregating them via `SUM(amount)` to determine the overall fee status.

- [x] **Real-Time Fee KPI Dashboard:** Four live KPI cards — Expected, Collected, Pending, and Collection Rate — update instantly based on the selected month and batch filter.

- [x] **Fee Collection Matrix:** A batch-grouped student list showing paid/partial/unpaid status with color-coded badges (✅ Paid, ⚠️ Partial, ⬜ Unpaid) and remaining balance calculations.

- [x] **Record Payment Modal:** A context-aware dialog pre-populated with the student's expected fee, already-paid total, and remaining balance — the admin just confirms and submits.

- [x] **Payment Undo & Audit Trail:** Each payment entry records the admin's email and timestamp. The most recent payment can be undone with a single click and confirmation dialog.

- [x] **Student Fee History:** An inline expandable accordion per student showing all payment records across all months — amount, date, and notes — fetched directly from the database.

### 📊 Visual Analytics & Exports

- [x] **Real-Time Chart Dashboard:** Four Chart.js visualizations — Enrollment Growth Timeline (line), Course Distribution (doughnut), Batch Distribution (doughnut), Gender Demographics (pie), and School Class Distribution (bar) — all rendered from live database aggregations.

- [x] **Professional Excel Exports:** One-click `.xlsx` generation via SheetJS with structured columns for both student data and fee reports, respecting the currently active filters.

- [x] **Institution-Branded PDF/Print:** Print-optimized layouts with the MQLC logo, title, and audit timestamp — hidden from the UI but auto-injected before `window.print()` for professional hardcopy directories.

- [x] **Fee Report Export:** Dedicated export functionality for fee data (separate from student exports) with columns for Expected, Paid, Remaining, and Status — filterable by batch.

### 🔒 Security & Infrastructure

- [x] **Row-Level Security (RLS):** Supabase RLS policies restrict anonymous access to `otp_pins` (SELECT/UPDATE only) and `quiz_leaderboard` (INSERT/SELECT only), while `fee_payments` is admin-only via authenticated policies.

- [x] **Cloudinary Media Pipeline:** Serverless API routes (`/api/get-bulletin`, `/api/get-updates`) fetch resources from Cloudinary using server-side credentials, never exposing API keys to the client.

- [x] **Vercel Edge Deployment:** JAMstack architecture with zero-config edge deployment, automatic HTTPS, and global CDN distribution for sub-100ms page loads.

- [x] **Environment Isolation:** All sensitive keys (Supabase, Cloudinary) are stored in `.env.local` and injected via Vercel's serverless runtime — never committed to the repository.

## 🏗️ Technical Architecture

### The Engine
- **Frontend:** Vanilla HTML5/CSS3/ES6+ (No heavy frameworks for 0ms hydration latency).
- **Backend-as-a-Service:** **Supabase** (Postgres) managing secure student records and authentication.
- **Data Ops:** **SheetJS** for XLSX generation and institutional **CSS Media Queries** for PDF layout.
- **Analytics:** **Chart.js** for high-performance client-side rendering.

### Static vs. Dynamic
| Component | Type | Technology |
| :--- | :--- | :--- |
| **Hero & Info Sections** | Static | Content-Optimized HTML |
| **Prayer Synchronization** | Dynamic | Aladhan API (Real-time sync) |
| **Student Matrix** | Dynamic | Supabase Real-time Queries |
| **Admin Analytics** | Dynamic | Chart.js Data Mapping |
